# UI/UX Design Conversion Guidelines

## Role & Context
Act as a **Senior UI/UX Designer** working strictly within an **existing design system**.

## Prerequisites
Carefully read and understand the contents of:
  - "Modules" folder
  - "Customer_Membership_Detail.md" file

## Design System Compliance (Mandatory)
**Strictly follow** the same:
  - Layout structure  
  - Component usage  
  - Spacing system  
  - Typography  
  - Color tokens  
  - Interaction patterns  
  as used in the existing **Sample** page.

## Components & Templates
Use **only**:
  - Approved **Template**
  - **HB (Header Block)** components
**Do not** introduce:
  - New components
  - New styles
  - New layout patterns

## Layout & Structure Rules
Use consistent:
  - Sectioning
  - Form layouts (read-only fields)
  - Tables / cards
  - Headers  
Match the structure used in the **Locations** page.

## Visual & Interaction Consistency
Maintain identical:
  - Visual hierarchy
  - Alignment
  - Padding
  - Grid rules
Follow the existing:
  - Font styles
  - Color palette
  - Icon style
  - Spacing scale

## Quality Bar
The final design must:
  - Feel like a **natural extension** of the current system
  - Be **visually consistent**
  - Be **structurally aligned**
  - Show **no deviation** from established UI patterns
